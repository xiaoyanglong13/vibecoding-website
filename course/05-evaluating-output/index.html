Module 5 · 75 minutes

Knowing when to trust the output

The skill that separates working vibecoders from frustrated ones. How to check if the AI actually did what you asked — and what to do when it didn't.

What you'll be able to do

Evaluate any AI-produced artifact for correctness, spot hallucinated functions and fabricated data, debug failures that happen silently, and know when to stop iterating and ask a real engineer.

Planned sub-pages

  • Checking correctness. Lightweight tests you can ask the agent to write for itself. Spot-checks that take 30 seconds.
  • Spotting hallucinations. Made-up API endpoints, invented column names, plausible-but-wrong numbers. Where they come from and how to catch them.
  • Debugging when it fails. Reading error messages, narrowing down the cause, telling the agent productively what went wrong.
  • When to stop and ask an engineer. Signals you've hit the edge of what vibecoding can responsibly do.
